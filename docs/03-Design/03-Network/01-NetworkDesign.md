# 네트워크 설계

## 개요

NMSIM은 다중 IP를 지원하여 각 가상 장비가 고유한 IP 주소를 가진다.
NMS에서는 실제 장비에 접근하듯이 각 IP로 SNMP 요청을 보낼 수 있다.

## 배포 환경

### VM Appliance

Docker의 다중 IP 제약으로 인해 **전용 VM**으로 배포한다.

```
┌─────────────────────────────────────┐
│ NMSIM VM Appliance                  │
│                                     │
│  OS: Ubuntu 22.04 LTS               │
│  사양: 2 vCPU, 4GB RAM, 20GB Disk   │
│                                     │
│  배포 형식:                          │
│  - OVA (VMware, VirtualBox)         │
│  - QCOW2 (KVM, Proxmox)             │
└─────────────────────────────────────┘
```

## 다중 IP 관리

### 동작 방식

```
┌─────────────────────────────────────────┐
│ NMSIM VM                                │
│                                         │
│   eth0: 관리용 IP (예: 192.168.1.50)    │
│                                         │
│   시뮬레이션 IP (자동 생성):             │
│   eth0: 10.0.0.1   (Router-01)          │
│   eth0: 10.0.0.2   (Router-02)          │
│   eth0: 10.0.0.41  (570FC-01)           │
│   eth0: 10.0.0.51  (570ACO-01)          │
│   eth0: 10.0.0.61  (570DSK-01)          │
│   ... (N개 장비)                        │
│                                         │
└─────────────────────────────────────────┘
```

### IP 할당/해제

```bash
# NMSIM 시작 시 - 등록된 장비 IP 일괄 생성
ip addr add 10.0.0.1/24 dev eth0
ip addr add 10.0.0.2/24 dev eth0
ip addr add 10.0.0.41/24 dev eth0
...

# 웹 UI에서 장비 추가 시 - 즉시 IP 생성
POST /api/v2/devices  { "ip_address": "10.0.0.100", ... }
  → ip addr add 10.0.0.100/24 dev eth0
  → 즉시 SNMP 응답 가능

# 장비 삭제 시 - IP 해제
DELETE /api/v2/devices/{id}
  → ip addr del 10.0.0.100/24 dev eth0

# NMSIM 종료 시 - 모든 IP 해제
ip addr del 10.0.0.1/24 dev eth0
...
```

## 네트워크 토폴로지

### 방법 A: 같은 대역 사용 (권장)

```
┌──────────────┐     ┌──────────────┐
│   NMS 서버   │     │  NMSIM VM    │
│  10.0.0.10   │     │  10.0.0.50   │
└──────┬───────┘     │              │
       │             │ + 10.0.0.1   │
       │             │ + 10.0.0.41  │
       │             │ + 10.0.0.51  │
       │             └──────┬───────┘
       │                    │
   ────┴────────────────────┴────
           10.0.0.0/24
```

- NMS와 NMSIM을 같은 대역에 배치
- 추가 라우팅 불필요

### 방법 B: 다른 대역 사용

```
┌──────────────┐     ┌──────────────┐
│   NMS 서버   │     │  NMSIM VM    │
│ 192.168.1.10 │     │ 192.168.1.50 │
└──────┬───────┘     │              │
       │             │ + 10.0.0.1   │
       │             │ + 10.0.0.41  │
       │             └──────┬───────┘
       │                    │
   ────┴────────────────────┴────
          192.168.1.0/24
```

NMS 서버에 라우팅 추가 필요:
```bash
ip route add 10.0.0.0/24 via 192.168.1.50
```
